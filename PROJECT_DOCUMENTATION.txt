1. PROJECT OVERVIEW
================================================================================

The b++ Web IDE is a complete web-based integrated development environment for
the b++ programming language (a Banglish-style toy language). The project
evolved from an initial compiler implementation into a full-stack web
application with:

- Interactive code editor with syntax highlighting
- Real-time code execution
- Help modal with language reference and copy example
- Download functionality for code files
- Responsive design for desktop and mobile
- Examples dropdown in header for quick loading
- One-click copy buttons in editor and help modal
- **TypeScript compiler backend** integrated with Next.js frontend (no Python dependency)

The project is optimized for deployment to Vercel using the default Node.js runtime; no
custom build commands or additional runtimes are required.

5. B++ LANGUAGE SPECIFICATION
================================================================================

B++ is a Banglish-style toy language with the following features:

KEYWORDS:
- dhori: Variable declaration (dhori x = 10;)
- dekhao: Print statement (dekhao x;)
- jodi: If condition (jodi x > 5 { ... })
- nahole: Else condition (nahole { ... })
- ghuri: For loop (ghuri i = 0; i < 10; i = i + 1 { ... })

OPERATORS:
- Arithmetic: + (addition), - (subtraction), * (multiplication), / (division)
- Comparison: > (greater), < (less), >= (greater-equal), <= (less-equal)
- Assignment: = (assign), == (equality comparison)
- Logical: && (and), || (or)

DATA TYPES:
- Numbers (integers and floats)
- Variables (identifiers)

PROGRAM STRUCTURE:
- Statements separated by semicolons or on new lines
- Comments: Single-line with //

EXAMPLE PROGRAM - SUM (0 to 9):
dhori s = 0;
ghuri i = 0; i < 10; i = i + 1 {
  s = s + i;
}
dekhao s;

OUTPUT: 45

EXAMPLE PROGRAM - FACTORIAL:
dhori n = 5;
dhori result = 1;
ghuri i = 1; i <= n; i = i + 1 {
  result = result * i;
}
dekhao result;

OUTPUT: 120

7. CRITICAL IMPLEMENTATION DETAILS
================================================================================

TYPESCRIPT COMPILER PIPELINE:

1. LEXER (lexer.ts)
  - Tokenizes raw b++ source code
  - Recognizes keywords (dhori, dekhao, jodi, nahole, ghuri)
  - Supports arithmetic and comparison operators, identifiers, numbers

2. PARSER (parser.ts)
  - Recursive descent parser with operator precedence
  - Builds AST for declarations, assignments, prints, if/else, for loops

3. INTERPRETER (interpreter.ts)
  - Executes AST within an environment map
  - Collects output for `dekhao` statements
  - Handles arithmetic and comparison operations (integers, floor divide)

4. MAIN ENTRY (main.ts)
  - `runCode(source: string): string`
  - Runs PHASE 1/2/3, formats output blocks and errors

API EXECUTION FLOW:

1. Frontend (page.tsx) sends POST to /api/run with `{ code }`
2. API Handler (route.ts) calls `runCode(code)` directly (no subprocess, no temp files)
3. Returns JSON `{ success, output, error }`
4. OutputPanel renders PHASE blocks, highlights errors in red

15. FILE-BY-FILE IMPLEMENTATION DETAILS
================================================================================

app/page.tsx:
- Main IDE component
- Manages state: code, output, error, loading, showInstructions
- Handles code execution via runCode() async function
- Renders header, editor, output panel, help panel
- Provides Run, Clear, Download, Help controls above editor
- Examples dropdown in header (and mobile in controls row)
- Copy buttons in editor header and help modal examples
- "use client" directive makes it a client component

app/api/run/route.ts:
- Next.js API route handler
- POST method accepts { code: string }
- Creates temporary .bpp file in /tmp or similar
- Spawns Python subprocess with working directory as compiler/
- Captures stdout and stderr
- Returns { success: boolean, output: string, error: string | null }
- Handles edge cases: empty code, execution timeout, file I/O errors

compiler/lexer.py:
- Defines TokenType enum with all token types
- Defines Token class with type and value
- Implements Lexer class with tokenize() method
- Handles keywords, operators, numbers, identifiers
- Returns list of Token objects
- No external dependencies

compiler/parser.py:
- Implements recursive descent parser
- Defines statement() method for parsing all statement types
- Implements expr(), term(), factor() for operator precedence
- Returns Program node containing list of statements
- Handles errors with descriptive messages
- Supports: VarDecl, AssignStmt, PrintStmt, IfStmt, ForStmt

compiler/ast_nodes.py:
- Defines AST node classes as Python dataclasses
- Node types: Program, VarDecl, AssignStmt, PrintStmt, IfStmt, ForStmt, 
             BinOp, UnaryOp, Num, Var
- Each node has relevant fields (name, value, left, right, condition, etc.)
- Used by both parser and interpreter

compiler/interpreter.py:
- Implements Interpreter class
- Maintains self.env dictionary for variable storage
- Maintains self.output list for print statements
- Implements exec() method for statement execution
- Implements eval() method for expression evaluation
- Handles all AST node types
- Returns formatted output with values

compiler/main.py:
- Entry point for Python execution
- Imports all compiler modules with proper sys.path handling
- Implements run_code(code) function
- Coordinates: Lexer → Parser → Interpreter pipeline
- Catches and formats errors
- Returns formatted output with phase information

components/CodeEditor.tsx:
- React component wrapping CodeMirror
- Props: value (code string), onChange callback
- Uses @codemirror/lang-cpp for syntax highlighting
- Styled with dark theme from globals.css
- Font size 16px for readability
- Responsive width

components/OutputPanel.tsx:
- React component displaying compiler output
- Props: output (string), error (string), loading (boolean)
- Parses output to extract "Program Output:" section
- Highlights actual output values in cyan-to-emerald gradient
- Shows loading spinner during execution
- Shows error messages in red

components/ExamplesMenu.tsx:
  - React dropdown button listing example programs
  - Contains 4 examples: If/Else, For Loop, Sum, Factorial
  - Props: onSelect callback to set editor code
  - Helps new users learn b++ syntax quickly from header

app/globals.css:
- Global CSS and Tailwind imports
- Custom CodeMirror styling:
  * Background: #0f172a (slate-950)
  * Text: #e2e8f0 (slate-100)
  * Font size: 16px
  * Line height: 1.5
  * Selection highlight: cyan overlay in editor and page
- Responsive design breakpoints

17. TROUBLESHOOTING GUIDE
================================================================================

PROBLEM: "npm install fails with missing dependencies"
SOLUTION:
- Clear npm cache: npm cache clean --force
- Delete node_modules and package-lock.json
- Run npm install again

PROBLEM: "Python: ModuleNotFoundError when running code"
SOLUTION:
- Check that sys.path includes compiler directory
- Verify compiler/ folder exists with all .py files
- Check main.py has: sys.path.insert(0, str(THIS_DIR))

PROBLEM: "CodeMirror not displaying or showing errors"
SOLUTION:
- Clear browser cache
- Check globals.css is loaded (inspect CodeMirror element)
- Verify @uiw/react-codemirror is installed
- Check no console errors in browser dev tools

PROBLEM: "Code doesn't execute and shows loading spinner forever"
SOLUTION:
- Check API route (app/api/run/route.ts) exists
- Verify Python path in API route is correct
- Check browser network tab for API response
- Look at Vercel logs for subprocess errors

PROBLEM: "Download creates empty or corrupt file"
SOLUTION:
- Check downloadCode() function in page.tsx
- Verify blob is created with text/plain mimetype
- Test with simple code first (no special characters)

PROBLEM: "Syntax highlighting not working"
SOLUTION:
- Verify @codemirror/lang-cpp is installed
- Check CodeEditor.tsx passes cpp extension
- Clear browser cache and reload

PROBLEM: "Help panel not showing"
SOLUTION:
- Check showInstructions state is initialized to false
- Verify button onClick toggles state correctly
- Check Instructions panel renders when showInstructions is true

PROBLEM: "Examples menu doesn't load programs"
SOLUTION:
- Check ExamplesMenu.tsx has all example definitions
- Verify onSelect callback is passed correctly to component
- Check page.tsx setCode(selectedCode) in handler
